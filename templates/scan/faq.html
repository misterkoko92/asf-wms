    {% trans "Vue d'ensemble" %}

    {% blocktrans trimmed %} Cette page décrit le fonctionnement complet du WMS Scan, du stock à l'expédition. Les flux sont conçus pour être rapides sur mobile et pour rester cohérents en base. {% endblocktrans %}

    • {% blocktrans trimmed %}Produit : fiche article (SKU, nom, marque, poids, emplacement).{% endblocktrans %}
    • {% blocktrans trimmed %}Lot : stock réel par produit (quantité, péremption, lot, emplacement).{% endblocktrans %}
    • {% blocktrans trimmed %}Carton : unité de préparation (DRAFT, PICKING, PACKED, ASSIGNED, LABELED, SHIPPED).{% endblocktrans %}
    • {% blocktrans trimmed %}Expédition : regroupe des cartons pour une destination.{% endblocktrans %}
    • {% blocktrans trimmed %}Suivi expédition : journal des scans (QR) avec horodatage.{% endblocktrans %}

    {% trans "Statuts cle" %}

    • {% blocktrans trimmed %}Carton : Créé (draft), En préparation (picking), Prêt (packed), Affecté (assigned), Étiqueté (labeled), Expédié (shipped).{% endblocktrans %}
    • {% blocktrans trimmed %}Expédition : Création (draft), En cours (picking), Prêt (packed), Planifié (planned), Expédié (shipped), Reçu escale (received_correspondent), Livré (delivered).{% endblocktrans %}
    • {% blocktrans trimmed %}Litige : drapeau transverse sur expédition (is_disputed), activable à tout moment du suivi.{% endblocktrans %}
    • {% blocktrans trimmed %}Réception : DRAFT tant que les lignes ne sont pas réceptionnées.{% endblocktrans %}

    {% trans "Créer une expédition" %}

    • {% trans "Séquencement: destination d'abord, puis expéditeur, puis destinataire + correspondant, puis le reste du formulaire." %}
    • {% blocktrans trimmed %}Expéditeur: tag Expéditeur + destination explicite ou globale (destinations vides).{% endblocktrans %}
    • {% blocktrans trimmed %}Destinataire: tag Destinataire + expéditeur lié explicite ou global (expéditeurs liés vides).{% endblocktrans %}
    • {% blocktrans trimmed %}Correspondant: tag Correspondant + destination; si la destination a un correspondant lié, ce choix est imposé.{% endblocktrans %}
    • {% trans "Si aucun correspondant n'est lié à la destination, le sélecteur correspondant reste vide." %}
    • {% blocktrans trimmed %}Ajouter des cartons Prêt non affectés, ou créer des cartons depuis des produits.{% endblocktrans %}
    • {% blocktrans trimmed %}À l'affectation à l'expédition, un carton passe en statut Affecté.{% endblocktrans %}
    • {% blocktrans trimmed %}Enregistrer en brouillon crée une expédition temporaire EXP-TEMP-XX modifiable plus tard.{% endblocktrans %}
    • {% blocktrans trimmed %}Créer des colis multi-produits (depuis la création) enregistre d'abord un brouillon puis ouvre l'onglet Préparation.{% endblocktrans %}

    {% trans "Vue Expéditions" %}

    • {% blocktrans trimmed %}Brouillon : expédition temporaire (référence EXP-TEMP-XX) sauvegardée pour reprise ultérieure.{% endblocktrans %}
    • {% blocktrans trimmed %}Création : expédition active sans colis étiqueté (référence finale).{% endblocktrans %}
    • {% blocktrans trimmed %}En cours : tant que tous les colis ne sont pas étiquetés.{% endblocktrans %}
    • {% blocktrans trimmed %}Prêt : tous les colis sont étiquetés.{% endblocktrans %}
    • {% blocktrans trimmed %}Planifié : verrouille la modification des colis de l'expédition.{% endblocktrans %}
    • {% blocktrans trimmed %}Expédié, Reçu escale, Livré : progression via le suivi.{% endblocktrans %}
    • {% blocktrans trimmed %}Litige : préfixe visuel sur le statut (ex: "Litige - Planifié").{% endblocktrans %}

    {% blocktrans trimmed %} La date mise en disponible est renseignée au passage en Prêt, puis effacée si l'expédition repasse en En cours. {% endblocktrans %}
